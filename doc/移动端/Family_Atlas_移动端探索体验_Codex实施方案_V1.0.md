# Family Atlas 移动端探索体验 — Codex 完整实施方案 V1.0

> 将本文档直接交给 Codex。本文中的每个 `MOB-*` ID 与《移动端探索体验实施清单 V1.0》一一对应。

## 0. 执行优先级与边界

1. 用户已确认决策。
2. PC 端最终冻结规范中的产品术语、CTA 状态机、数据语义和品牌视觉。
3. 本移动端实施方案。
4. 现有仓库代码；临时占位实现不得覆盖冻结规范。

不得重新讨论或修改 PC 已冻结的热榜、适配度、Journey、心愿、点亮、同步等业务定义。移动端只实现适合触摸设备的呈现和交互。

## 1. 开工前仓库审计

Codex 在修改代码前必须先输出影响分析：

- 找出世界、洲、国家、地区、POI 详情的路由与入口。
- 找出地图/地球 SDK、地图实例创建位置、相机 API、Marker/Cluster API。
- 找出状态管理、路由状态、缓存、同步和离线写入层。
- 找出现有 Header、Bottom Sheet、Card、CTA、Search、Filter、Toast 组件。
- 找出测试框架、Storybook、E2E、视觉回归与无障碍测试能力。
- 列出计划新增、修改、删除的文件，不得先重写后说明。
- 优先复用现有技术栈，不得在仓库中再造平行状态管理或第二套地图框架。

## 4. 逐项实施要求

### P0 任务

#### MOB-P0-001｜建立统一 MobileAtlasShell，世界、洲、国家、地区探索共用同一移动端结构

**采用方案：** GPT + DeepSeek + Claude

**实现：** 新增 `MobileAtlasShell` 作为唯一探索页外壳，内部固定包含 `CompactHeader`、`AtlasStage`、`MapFloatingTools`、`AtlasBottomSheet`、`ConditionalPrimaryAction`。各层级只注入地图数据、上下文摘要、筛选与列表内容，不复制页面骨架。

**验收：** 四个探索层级的 DOM/组件骨架一致；切层不出现结构闪烁；共享布局修复只需改一处。

#### MOB-P0-002｜页面统一采用“紧凑顶部栏 + 全屏地图/地球 + Bottom Sheet + 条件式 CTA”

**采用方案：** 全模型共识

**实现：** 移除旧版地图上方/下方纵向堆叠布局。地图作为全屏底层，Header、工具和 Sheet 采用 overlay；CTA 固定在 Sheet 内部底部区域。

**验收：** 探索页不存在固定高度小地图、地图下方长空白或独立悬浮 CTA。

#### MOB-P0-003｜保持同一地图实例或可恢复相机实例，避免切层重复初始化

**采用方案：** Claude

**实现：** 将地图实例提升至 Shell 或路由级 Provider；层级切换仅更新数据源、边界与相机目标。若技术栈必须重建，先序列化相机、图层、选中状态并在首帧恢复。

**验收：** 连续切换世界→洲→国家→地区时无白屏、无默认视角闪回；WebGL Context 数量稳定。

#### MOB-P0-004｜地图使用 100dvh 与 VisualViewport 动态铺满可用屏幕

**采用方案：** GPT + DeepSeek + Grok

**实现：** Web 端优先使用 `100dvh`，监听 `visualViewport.resize` 处理地址栏和软键盘；原生容器使用安全区后的可用高度。禁止以 `100vh` 或固定像素撑高页面。

**验收：** iOS Safari 地址栏展开/收起和 Android 浏览器滚动时，地图与 Sheet 不跳动、不产生额外空白。

#### MOB-P0-005｜禁止 Body 长页面滚动，探索交互仅由地图、抽屉和抽屉内容各自处理

**采用方案：** GPT + Claude 补全

**实现：** Shell 设置 `overflow:hidden`、`overscroll-behavior:none`；地图层、Sheet 拖动层和 Sheet 内容滚动容器分别管理手势。禁止通过页面整体滚动连接地图与列表。

**验收：** 页面本体不出现浏览器级长滚动；地图和列表之间无大段空白。

#### MOB-P0-006｜Bottom Sheet 定义为 peek / browse / detail 三个语义档位

**采用方案：** GPT

**实现：** 建立 `SheetDetent = 'peek' | 'browse' | 'detail'`，不在业务代码直接使用百分比。设计参考范围：peek 20%–30%、browse 48%–62%、detail 85%–94%。

**验收：** 代码中不存在硬编码 `25vh/55vh/90vh` 作为业务档位判断。

#### MOB-P0-007｜抽屉高度根据视口、安全区、CTA、动态字体和键盘状态计算

**采用方案：** GPT + Claude

**实现：** 创建 `resolveSheetHeight(detent, layoutContext)`，输入可用高度、安全区、CTA 高度、字体缩放、键盘高度、内容最小高度；输出合法像素高度。

**验收：** 200% 动态字体、软键盘、刘海屏和横屏下均不裁切主要内容与 CTA。

#### MOB-P0-008｜为三个档位定义内容契约

**采用方案：** Claude + GPT

**实现：** peek 仅展示上下文标题、结果数和当前选中摘要；browse 展示快速筛选、主列表和核心操作；detail 展示完整档案、深度筛选、适配度解释和来源。使用条件渲染而非仅改变高度。

**验收：** peek 不出现长列表或长文；browse 不堆满完整档案；detail 信息结构完整。

#### MOB-P0-009｜按页面和进入路径动态决定初始档位

**采用方案：** GPT

**实现：** 建立 `resolveInitialDetent(entryContext)`：首页/直接进入区域→peek；搜索、推荐、点位选中、Journey 首页→browse；完整筛选、长详情→detail。

**验收：** 不同入口的初始档位符合规则，刷新和深链有稳定默认值。

#### MOB-P0-010｜用户主动调整的抽屉档位在当前页面会话中保持

**采用方案：** GPT

**实现：** 将 detent 放入 `AtlasViewState`；数据刷新、地图移动和短暂切后台不得重置；仅路由上下文变化或用户操作改变。

**验收：** 用户拖到 detail 后切换筛选、同步或短暂离开应用，返回仍保持合理档位。

#### MOB-P0-011｜定义地图净可视区与相机避让规则

**采用方案：** Claude + GPT

**实现：** 为每个 detent 计算 `cameraPadding`。peek 时地图裸露区建议≥65%，browse 建议≥35%；选中点位定位到裸露区中心偏上，避开 Header、Sheet、CTA、地图工具。

**验收：** 任意档位下选中点位不被遮挡；相机定位后无需用户再次拖图寻找目标。

#### MOB-P0-012｜地图点位单击立即选中

**采用方案：** GPT

**实现：** 单击未选中 Marker 立即设置 `selectedEntityId`，显示摘要，Sheet 升至 browse，列表滚动到对应项；废除首次选中需二次点击的流程。

**验收：** 一次点击完成选中；不会与地图双击缩放冲突。

#### MOB-P0-013｜再次点击已选中点位或查看详情按钮进入完整详情

**采用方案：** GPT

**实现：** 已选中 Marker 再次点击时升级至 detail 或进入 POI 路由；摘要卡同时提供明确“查看详情”按钮，避免依赖隐藏手势。

**验收：** 用户可通过再次点击或可见按钮进入详情，两条路径结果一致。

#### MOB-P0-014｜点击其他点位直接切换选中对象

**采用方案：** GPT + DeepSeek

**实现：** 新点位选择应取消旧定位动画、恢复旧 Marker、更新唯一选中状态，再定位新对象；不得先要求取消旧对象。

**验收：** 快速连续点击多个点位时最终状态只对应最后一次选择。

#### MOB-P0-015｜点击地图空白区域取消选中并恢复合理档位

**采用方案：** GPT

**实现：** 空白点击清除选中与摘要；若 Sheet 是因选中对象自动升至 browse，则回到此前档位；若用户主动升档则保留用户档位。

**验收：** 取消选中不会机械破坏用户手动选择的 Sheet 状态。

#### MOB-P0-016｜地图双击放大仅在空白区域生效，长按不绑定核心功能

**采用方案：** GPT

**实现：** Marker 事件阻止双击传播；空白双击执行地图放大。长按只保留平台默认或未来扩展，不承载首发关键操作。

**验收：** 点位选择、双击放大和长按互不冲突。

#### MOB-P0-017｜地图与列表共享唯一选中和聚焦状态源

**采用方案：** Claude + GPT

**实现：** 在统一 Store 中维护 `selectedEntityId`、`focusedEntityId`、`comparedEntityIds`；地图和列表只订阅同一状态，不在组件内维护平行选择状态。

**验收：** 地图、列表、摘要和 CTA 不出现选中对象不一致。

#### MOB-P0-018｜点击列表项或“在地图中查看”时才执行地图定位

**采用方案：** GPT

**实现：** 列表卡点击更新 selected 并调用可取消的 `flyToEntity`；普通列表进入视口或滚动不触发相机移动。

**验收：** 列表滚动稳定，只有明确选择动作改变地图中心。

#### MOB-P0-019｜点击地图点位后列表自动滚动并高亮

**采用方案：** Claude + GPT

**实现：** 通过实体 ID 查找列表索引并调用虚拟列表 `scrollToItem`；增加来源标记避免滚动回调再次触发地图定位造成循环。

**验收：** 地图→列表联动准确，无循环抖动。

#### MOB-P0-020｜普通纵向列表滚动不得持续驱动地图移动

**采用方案：** GPT

**实现：** 移除基于当前可见卡片自动 flyTo 的逻辑；可仅更新轻量 focus 样式，不改变相机中心。

**验收：** 快速滚动列表时地图保持稳定，不出现晕动和动画队列。

#### MOB-P0-021｜选中后其他点位降至 40%–55% 透明度但仍可点击

**采用方案：** Claude + GPT

**实现：** 通过统一样式状态降低非选中 Marker 的 opacity/scale，保留 pointer hitbox 和可访问标签。

**验收：** 聚焦清晰，同时能直接切换到其他点位。

#### MOB-P0-022｜主决策列表全部纵向单列，横向只用于有限场景

**采用方案：** DeepSeek + GPT + Grok

**实现：** 亚洲国家、日本地区、POI、准备与风险列表使用单列；横向滚动仅允许首页推荐、图片媒体和 P2 对比。

**验收：** 主决策内容不被横向裁切，单手扫读顺畅。

#### MOB-P0-023｜同屏最多显示一张完整地图操作卡

**采用方案：** Claude + GPT

**实现：** Marker Layer 只允许 selected entity 渲染完整卡；其他对象最多显示点位或短标签。

**验收：** 密集地区无多张完整卡互相覆盖。

#### MOB-P0-024｜实现移动端标签 LOD、聚类、缩写与优先级隐藏

**采用方案：** Claude + DeepSeek

**实现：** 按缩放级渲染：远景点位/区域名，中景图标+≤4字简称，近景摘要；冲突时按 selected > recommendation > visited > normal > low 的优先级缩写、偏移、聚类或隐藏。

**验收：** 选中卡不裁切；普通标签允许按规则隐藏，不追求全部同时可见。

#### MOB-P0-025｜视口边缘与控件区域建立标签安全区

**采用方案：** Claude

**实现：** 基础边缘安全区不少于 8px，并叠加 Header、Sheet、工具按钮的动态 exclusion rect；标签布局引擎不得将卡片放入这些区域。

**验收：** 常见尺寸和缩放级下无选中卡越界或被工具遮挡。

#### MOB-P0-026｜聚类点点击后放大拆分，无法拆分时展示聚类列表

**采用方案：** Claude

**实现：** Cluster click 优先 fitBounds/zoom；到最大缩放仍聚合时，在 browse Sheet 内显示该簇的纵向列表。

**验收：** 所有聚类对象均可访问，不出现无响应的聚类点。

#### MOB-P0-027｜地图定位动画可被新操作取消

**采用方案：** Claude + GPT

**实现：** 每次 flyTo 分配 request token 或 AbortController；新选择、拖图、返回操作立即取消旧动画和旧回调。

**验收：** 快速操作后不会跳回过期目标。

#### MOB-P0-028｜地图加载或坐标失败时列表仍可完成核心流程

**采用方案：** Claude

**实现：** 地图异常切换为可重试占位，但保持 Sheet、列表、筛选、详情和 CTA 可用；提供“仅查看列表”和“重新加载地图”。

**验收：** 地图 SDK/瓦片失败不阻断探索、收藏与详情查看。

#### MOB-P0-029｜抽屉把手区只负责拖动抽屉

**采用方案：** GPT + Gemini

**实现：** 为视觉把手创建至少 44px 高的透明热区；该区域拦截纵向拖动，不滚列表、不操作地图。

**验收：** 把手拖动稳定，无地图或列表串扰。

#### MOB-P0-030｜抽屉内容区优先滚动列表，到顶后继续下拉才接管抽屉

**采用方案：** GPT + Grok + Gemini

**实现：** 使用嵌套滚动协调器：`scrollTop > 0` 时内容消费；`scrollTop == 0` 且下拉超过阈值时将控制权交给 Sheet。

**验收：** 无手势死锁、跳动和意外下沉。

#### MOB-P0-031｜地图裸露区负责单指平移、双指缩放和空白双击放大

**采用方案：** GPT + 千问局部建议

**实现：** 仅地图未被 Sheet 覆盖的区域接收地图手势；Marker 和工具控件阻止事件冒泡。

**验收：** 地图手势自然，点位与控件操作不会拖动地图。

#### MOB-P0-032｜横向媒体区域锁定水平方向并与纵向滚动正确交接

**采用方案：** GPT + Gemini 修正版

**实现：** 设置方向判断阈值；明显横向意图由 Carousel 消费，明显纵向意图交还 Sheet 内容滚动；手势结束后释放锁。

**验收：** 横滑推荐卡不移动地图，斜向手势不会同时触发两个容器。

#### MOB-P0-033｜CTA、筛选标签和按钮阻止手势穿透

**采用方案：** Claude

**实现：** 交互控件在 pointer/touch start 时阻止地图和 Sheet 拖动识别；点击结束后正常执行按钮行为。

**验收：** 点击 CTA 不会同时拖动抽屉或地图。

#### MOB-P0-034｜所有核心触摸目标≥44×44px，相邻小按钮间距≥8px

**采用方案：** 千问 + DeepSeek + GPT + Claude

**实现：** 以 hit area 而非仅视觉尺寸验收；对小图标添加透明 padding，地图 Marker 也使用至少 44px 触摸热区。

**验收：** 自动尺寸检查和真机误触测试通过。

#### MOB-P0-035｜顶部采用返回、当前层级触发器和最多两个工具按钮

**采用方案：** GPT

**实现：** 构建 `CompactHeader`：左返回，中间当前层级按钮，右搜索/菜单；移除常驻完整面包屑与英文 BACK。

**验收：** 顶部高度稳定，标题不被工具挤压，核心控件均可触达。

#### MOB-P0-036｜路径菜单支持跨级跳转并恢复焦点

**采用方案：** Claude + GPT

**实现：** 当前层级点击打开轻量 Popover/小型 Sheet，展示世界→洲→国家→地区路径；关闭后焦点回到触发器。

**验收：** 深链进入也能正确返回和跨级跳转。

#### MOB-P0-037｜探索界面不增加固定四 Tab，产品级入口收进顶部菜单

**采用方案：** GPT + DeepSeek + Grok + Gemini

**实现：** 删除旧底部“探索/足迹/心愿/旅程”导航；地图情境图层通过工具菜单提供，全局模块通过菜单或产品外壳进入。

**验收：** 底部仅保留 Sheet/CTA，不与 Home Indicator 争抢空间。

#### MOB-P0-038｜搜索使用全屏覆盖层或独立路由

**采用方案：** GPT

**实现：** 建立 `MobileSearchOverlay`，包含输入框、最近浏览、建议、分组结果、空状态；软键盘出现时使用 VisualViewport 保持输入可见。

**验收：** 搜索不挤压地图和 Sheet，不出现键盘遮挡。

#### MOB-P0-039｜搜索打开时暂停地图输入，关闭后恢复原现场

**采用方案：** GPT

**实现：** 打开搜索时禁用地图 pointer input，但无需销毁地图；保存并恢复 `AtlasViewState` 和焦点。

**验收：** 取消搜索后地图、选中、筛选和 Sheet 与打开前一致。

#### MOB-P0-040｜返回遵循分层状态机

**采用方案：** GPT + Grok + Gemini

**实现：** 统一 reducer：关闭全屏筛选→关闭深度详情→取消选中→detail 降 browse→browse 降 peek→返回上一空间层级。系统返回、侧滑返回和 UI 返回调用同一 action。

**验收：** 任意中间状态按预期逐层退出，不一键离开页面。

#### MOB-P0-041｜建立 AtlasViewState 快照并无损恢复探索现场

**采用方案：** GPT + Gemini + Claude

**实现：** 至少保存 center、zoom、bearing/pitch、detent、filters、sort、listScrollTop、selectedEntityId、sync snapshot；进入 POI 前写入，返回时恢复。

**验收：** 从 POI 返回后位置、缩放、筛选、滚动和选中对象完全恢复。
