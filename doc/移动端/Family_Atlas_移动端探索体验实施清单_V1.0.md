# Family Atlas 移动端探索体验实施清单 V1.0

> 状态：最终施工基线。PC 端已冻结的产品术语、CTA 状态机、数据语义和视觉职责直接继承，本清单只定义移动端布局、交互、可用性与工程验收。
>
> 最终采用结构：GPT 负责移动交互主裁决；Claude 负责工程骨架、可访问性与验收红线；DeepSeek/Grok 负责结构边界；Gemini 负责降级后的触感与 AI 增强；千问仅采纳局部触摸与缩放建议。

## 完成口径

- P0 全部完成后，才可进入移动端首发验收。
- P1 中测试、可访问性和异常降级不是可选美化项，必须随首发实现。
- P2 不阻塞首发，但代码结构不得阻止后续接入。
- 每个清单 ID 必须在 Codex 实施报告中标记：完成、延期或阻塞，并附证据。
